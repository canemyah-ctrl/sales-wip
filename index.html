How to host this on GitHub Pages
Create a GitHub account (if you don't have one)
Go to github.com and sign up for free. A free account gives you unlimited public repositories and free hosting via GitHub Pages.
Create a new repository
Click the + icon (top right) → New repository. Name it something like sales-wip. Set it to Public. Check "Add a README file". Click Create repository.
Upload your HTML file
Inside your new repo, click Add file → Upload files. Drag your notion-wip-theme.html file in. Scroll down and click Commit changes.

Important: rename the file to index.html before uploading — GitHub Pages serves index.html as the homepage automatically.
Enable GitHub Pages
Go to your repo → Settings → scroll to Pages in the left sidebar. Under Source, select Deploy from a branch. Set branch to main and folder to / (root). Click Save.
⏱ Wait ~60 seconds for GitHub to build your site
Access your live URL
GitHub Pages will give you a URL like: https://YOUR-USERNAME.github.io/sales-wip/ Share this with your sales team — they can open it in any browser.
Update the file each week
Go back to your repo → click index.html → click the pencil ✏️ icon to edit in browser. Or, re-upload a new version via Add file → Upload files (overwrite the existing one). Changes go live within seconds of committing.
Add your images to the repo (optional)
Create an images/ folder in the repo and upload your sales screenshots there. Then reference them in the HTML like: <img src="images/sale-w10-client1.png" /> This is the most reliable way to host images alongside your page — no external links needed.
